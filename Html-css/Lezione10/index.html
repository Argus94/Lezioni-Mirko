<i> vs <em> - <b> vs <strong>

La differenza importante sta nel fatto che <i> dà solo lo stile italic, <em> invece dà, oltre all'italic anche un significato di enfasi al testo, lo enfatizza. Stesso discorso vale per <b> e <strong>, uno dà solo lo stile, l’altro, oltre a dare il grassetto, fornisce pure un significato di forte importanza.

Cos'è l'identantion e perchè è importante

L'identantion o identazione è quel processo che sistema i vari elementi per ordine di importanza. Quando parliamo di lista e sottolista, ad esempio, i vari elementi di queste liste, comunemente vengono identate, ovvero vengono disposte da sinistra verso destra, in posizioni variabili in base all'importanza.
Si può attuare un'identazione con il plugin per atom che si chiama atom-beautify.
Perchè è importante? L'identazione del codice permette di individuare facilmente ad occhio quali elementi sono inclusi in certi tag e come essi si ramificano. Risulta quindi essenziale identare il codice quando si ha una pagina troppo complessa.

Quindi perchè il CSS è importante?

Quando ancora non esisteva il css la maggior parte degli oggetti venivano disposti su schermo tramite l'uso delle tabelle ed ecco che si venivano a creare tabelle infinite dove il rischio di sbagliare era molto alto, il codice diventava infinitamente lungo e si appesantiva inutilmente la pagina. Ecco che una delle esigenze che ha portato alla nascita del CSS è stata proprio quella di snellire il codice.
Oggi quindi si è iniziato ad adottare il css o cascading style sheet, che permette di dare un layout ed uno stile complesso e più dettagliato senza il rischio di andare a scrivere pagine lunghissime di codice html, racchiudendo tutti in classi con all’interno le varie proprietà con i corrispondenti valori, che successivamente vengono richiamati dalla pagina html tramite classi, id o gli stessi tag.

Come si deve studiare il CSS

Il CSS non è come l'HTML, non sono come i tag, pochi e specifici.
Il fattore chiave per saper usare il CSS è sfruttare il web, i motori di ricerca, ed impararlo ad utilizzare con la pratica.
Questo processo si può spiegare con un esempio:

Voglio rendere questo div di colore verde, come faccio?

Vado su google e cerco "div colored mdn, o background color div, o changing background color" il primo risultato che uscirà sarà proprio la proprietà css di qui avrò bisogno, ovvero background-color"

Ed ecco che aggiungendo la proprietà che ho cercato, mostra il colore di cui ho bisogno.

Una delle più importanti skill di uno sviluppatore è quella di riuscire ad uscire fuori da una situazione di stasi attraverso le giuste ricerche tramite motori di ricerca e documentations online. Quindi, quando si è bloccati, bisogna cercare ed utilizzare internet il più possibile, cosa quindi importante l’inglese!

La struttura del CSS

Come ben sappiamo in un foglio di stile troviamo questo format testuale:

Selettore { Proprietà: valore; }

  • Il selettore risponde alla domanda "Who?" ovvero a chi stiamo facendo riferimento.
  • La proprietà risponde alla domanda "What?" con cosa stiamo risponendo.
  • Il valore risponde alla domanda "How?" ovvero come stiamo rispondendo.

Perché è importante rispettare la punteggiature? Se andiamo a rimuovere i 2 punti, non si riuscirà a distinguere il cosa dal come;
Se dimentichi il punto e virgola, Non si potrà distinguere il come della precedente proprietà, dal “cosa” della successiva.
PS: È buon uso mettere i vari selettori in ordine alfabetico.

Il debugging e l'ordine di stile

Il debugging è la classica operazione di revisione per togliere eventuali errori detti in inglese bug.
Il debugging quindi è importante per fixare i bug presenti nelle pagine. Per riconoscere possibili bug è necessario aprire la console su chrome o firefox e vedere che errore viene riportato, dopo di che bisogna imparare a leggerli. Basta cercarli se non si capiscono.
Oltre agli errori nella console, sono anche presenti errori di css anche detti errori di priorità. Infatti se dovessimo inserire un parametro css in un foglio di stile esterno e successivamente inserire un parametro css dentro il foglio html, quello dentro quest’ultimo avrà la priorità e sovrascriverà quello nel foglio di stile.
L'ordine di priorità quindi è importantissimo poichè un parametro uguale invaliderebbe l'altro in ordine gerarchico. Per questo quando si importa un parametro da un foglio css utilizzato per più pagine e si vuole portare qualche accorgimento, si attuano delle modifiche in pagina html.
